# 故障排查指南

## 服务启动失败

### 现象：SERVICE_PAUSED

```
nssm start PyTaskSched
PyTaskSched: START: 操作成功完成。
nssm status PyTaskSched
SERVICE_PAUSED
```

### 诊断步骤

#### 1. 查看错误日志

```bash
type logs\scheduler_stderr.log
```

**常见错误信息**：

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Unable to create process` | Python 路径错误或不可访问 | 使用全局 Python `C:\Python314\python.exe` |
| `ModuleNotFoundError: No module named 'xxx'` | 依赖未安装 | `python -m pip install apscheduler loguru pyyaml` |
| `Permission denied` | 文件权限问题 | 检查日志目录权限 |

#### 2. 手动测试命令

```bash
# 测试调度器是否能运行
python -m src.scheduler --list

# 应该显示任务列表，如果报错说明代码或环境有问题
```

#### 3. 检查服务配置

```bash
# 检查使用的 Python 路径
nssm get PyTaskSched Application

# 检查参数
nssm get PyTaskSched AppParameters

# 检查工作目录
nssm get PyTaskSched AppDirectory

# 检查服务账户
nssm get PyTaskSched ObjectName
```

**预期结果**：
```
Application: C:\python314\python.exe
AppParameters: -m src.scheduler
AppDirectory: F:\github\person_project\windows-schedule
ObjectName: LocalSystem
```

#### 4. 检查 Python 安装

```bash
# 检查 Python 版本和路径
python --version
where python

# 应该显示: C:\Python314\python.exe
```

如果不是 `C:\Python314\python.exe`，需要重新安装全局 Python。

### 解决方案

#### 方案 1：重新安装全局 Python

1. 卸载现有 Python
2. 重新安装，**必须勾选**：
   - ✅ "Install Python for all users"
   - ✅ "Add Python to PATH"
   - 安装路径：`C:\Python314`

#### 方案 2：重新配置服务

```bash
# 删除旧服务
sc delete PyTaskSched

# 重新安装
scripts\install.bat

# 启动服务
nssm start PyTaskSched

# 验证状态
nssm status PyTaskSched
```

## 任务执行失败

### 现象：服务运行正常，但任务执行报错

#### 查看任务执行日志

```bash
type logs\scheduler.log
```

#### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `No such file or directory` | 项目路径不存在 | 检查 `tasks/*.yaml` 中的 `project_path` |
| `python.exe: not found` | 目标项目的 Python 路径错误 | 检查 `venv_path` 配置 |
| `ModuleNotFoundError` | 目标项目依赖缺失 | 在目标项目中安装依赖 |

## 服务无法停止

### 现象

```bash
nssm stop PyTaskSched
# 命令卡住或报错
```

### 解决方案

```bash
# 强制终止进程
taskkill /F /IM python.exe

# 然后再停止服务
nssm stop PyTaskSched
```

## 修改配置后不生效

### 原因

服务需要重启才能加载新配置。

### 解决方案

```bash
nssm restart PyTaskSched
```

## 日志文件过大

### 现象

日志文件占用过多磁盘空间。

### 解决方案

日志已配置自动轮转：
- 单文件最大 100 MB
- 保留 30 天

如需调整，修改 `src/scheduler.py` 中的配置：

```python
logger.add("logs/scheduler.log",
           rotation="100 MB",
           retention="30 days")
```

## 端口被占用

### 现象

如果调度服务需要监听端口（当前版本不需要），可能遇到端口冲突。

### 诊断

```bash
netstat -ano | findstr :端口号
```

### 解决方案

1. 停止占用端口的进程
2. 或修改配置使用其他端口

## 完全重置

如果遇到无法解决的问题，可以完全重置：

```bash
# 1. 停止并删除服务
nssm stop PyTaskSched
nssm remove PyTaskSched confirm

# 2. 删除日志（可选）
del logs\*.log

# 3. 重新安装
scripts\install.bat

# 4. 启动服务
nssm start PyTaskSched

# 5. 验证
nssm status PyTaskSched
```

## 获取帮助

如果以上方法都无法解决问题：

1. 收集以下信息：
   - 错误日志 (`logs\scheduler_stderr.log`)
   - 服务配置 (`nssm get PyTaskSched *`)
   - Python 版本 (`python --version`)

2. 检查 [技术变更记录](./技术变更记录.md) 了解已知问题

3. 参考 [部署指南](./部署指南.md) 确认环境配置正确
