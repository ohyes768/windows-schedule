# 部署指南

## 环境要求

- Windows 操作系统
- Python 3.14+ (全局安装)
- NSSM (Non-Sucking Service Manager)

## 安装步骤

### 1. 安装全局 Python

**重要**：必须安装全局 Python，服务才能正常运行。

1. 访问 https://www.python.org/downloads/ 下载 Python 3.14
2. 运行安装程序，**关键设置**：
   - 勾选 "Add Python to PATH"
   - 点击 "Customize installation"
   - 勾选 **"Install Python for all users"**
   - 勾选 "Add Python to environment variables"
   - **Customize install location** 改为：C:\Python314
3. 验证安装：
   python --version
   where python
   # 应该显示: C:\Python314\python.exe

### 2. 安装项目依赖

cd F:\github\person_project\windows-schedule
python -m pip install apscheduler loguru pyyaml

### 3. 下载并配置 NSSM

1. 访问 https://nssm.cc/download
2. 下载最新版本
3. 解压并将 nssm.exe 添加到系统 PATH，或复制到项目目录

### 4. 安装服务

scripts\install.bat

### 5. 启动服务

nssm start PyTaskSched

### 6. 验证服务状态

nssm status PyTaskSched
# 正常状态: SERVICE_RUNNING

## 常见问题

### 问题 1: SERVICE_PAUSED 状态

现象：
nssm start PyTaskSched
PyTaskSched: START: 操作成功完成。
nssm status PyTaskSched
SERVICE_PAUSED

原因：服务无法创建 Python 进程。

解决方案：

1. **检查 Python 安装**：必须是全局安装 (C:\Python314\python.exe)，不能是用户目录下的版本
2. **检查依赖安装**：确保已安装所有依赖到全局 Python
   python -m pip list | grep -E "loguru|apscheduler|pyyaml"
3. **查看错误日志**：
   type logs\scheduler_stderr.log

### 问题 2: ModuleNotFoundError

现象：
ModuleNotFoundError: No module named 'loguru'

原因：依赖未安装到全局 Python，或服务使用了错误的 Python。

解决方案：
# 确认使用全局 Python 安装依赖
python -m pip install apscheduler loguru pyyaml

### 问题 3: 虚拟环境无法运行

原因：Windows 服务默认使用 LocalSystem 账户，无法访问用户目录下的虚拟环境。

解决方案：使用全局 Python，不要使用虚拟环境。

## 服务管理命令

| 操作 | 命令 |
|------|------|
| 启动 | nssm start PyTaskSched |
| 停止 | nssm stop PyTaskSched |
| 重启 | nssm restart PyTaskSched |
| 状态 | nssm status PyTaskSched |
| 编辑配置 | nssm edit PyTaskSched |
| 删除服务 | nssm remove PyTaskSched confirm |

## 配置任务

在 tasks/ 目录创建 YAML 配置文件。

### 单个执行时间（兼容旧配置）

id: "my_task"
name: "我的任务"
description: "任务描述"
enabled: true
cron: "0 2 * * *"  # 每天凌晨 2 点
project_path: "F:/projects/my-project"
working_dir: "F:/projects/my-project"
script: "main.py"
args:
  - "--arg1"
  - "value1"
max_retries: 0
retry_delay: 60

### 多个执行时间（新功能）

id: "my_task"
name: "我的任务"
description: "任务描述"
enabled: true
crons:
  - "0 9 * * *"    # 每天 9:00
  - "0 18 * * *"   # 每天 18:00
project_path: "F:/projects/my-project"
working_dir: "F:/projects/my-project"
script: "main.py"
args:
  - "--arg1"
  - "value1"
max_retries: 0
retry_delay: 60

### 环境变量（可选）

# 对于需要使用 Playwright 等工具的任务，需要指定浏览器路径
env:
  PLAYWRIGHT_BROWSERS_PATH: "C:\Users\{YourUsername}\AppData\Local\ms-playwright"

**环境变量说明**：
- PLAYWRIGHT_BROWSERS_PATH：指定 Playwright 浏览器安装路径
- Windows 服务运行在 LocalSystem 账户下，无法自动访问用户目录
- 必须显式指定当前用户的浏览器路径

## Cron 表达式格式

分钟 小时 日期 月份 星期
*    *    *    *    *

| 字段 | 取值范围 | 示例 |
|------|----------|------|
| 分钟 | 0-59 | 0, 30, */15 |
| 小时 | 0-23 | 0, 9, 18 |
| 日期 | 1-31 | 1, 15, * |
| 月份 | 1-12 | 1, 6, * |
| 星期 | 0-6 (0=周日) | 1-5, * |

**常用示例**：
- 0 2 * * * - 每天凌晨 2 点
- 30 8 * * 1-5 - 周一到周五 8:30
- 0 */6 * * * - 每 6 小时
- 0 9 * * 1 - 每周一 9:00

## 技术要点

### NSSM 参数传递方式

错误方式：
nssm install ServiceName "python.exe" "-m module"

正确方式：
nssm install ServiceName "C:\Python314\python.exe"
nssm set ServiceName AppParameters "-m module"
nssm set ServiceName AppDirectory "C:\path\to\project"

### 服务账户

- 默认使用 **LocalSystem** 账户
- LocalSystem 无法访问用户目录下的 Python
- 解决方案：安装全局 Python 到 C:\Python314
